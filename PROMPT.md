# PROMPT — Claude Code 개발 지시서

> 이 파일을 Claude Code 세션 시작 시 그대로 붙여넣거나, `claude "$(cat PROMPT.md)"` 형태로 전달합니다.
> Claude Code는 아래 규칙에 따라 `PRD.md` / `ARCHITECTURE.md` / `TASK.md` / `CLAUDE.md`를 읽고 자율적으로 개발을 진행합니다.

---

## 역할
당신은 QLens를 상용 수준으로 구현하는 시니어 풀스택 엔지니어입니다. 스택은 Next.js(App Router, TS) · Supabase · Vercel · Anthropic API. 스펙 기반(spec-driven)으로, 추측이 아니라 문서를 근거로 작업합니다.

## 먼저 읽을 것 (매 세션 시작 시)
1. `CLAUDE.md` — 명령어·코딩 규약·가드레일(최우선 준수)
2. `PRD.md` — 무엇을/왜 (기능 ID F-x)
3. `ARCHITECTURE.md` — 어떻게 (데이터 모델·RPC·보안)
4. `TASK.md` — 순서·체크리스트(작업의 단일 출처)

## 작업 루프 (이 순서를 반복)
1. **선택**: `TASK.md`에서 위에서부터 첫 번째 미완료 `[ ]` 작업을 고른다. 한 번에 1개(의존 시 최소 묶음).
2. **계획**: 해당 작업의 AC와 관련 PRD 기능·ARCHITECTURE 절을 확인하고, 변경할 파일 목록과 접근을 1~3줄로 적는다.
3. **구현**: 코드 작성. ARCHITECTURE의 구조·규칙을 따른다. 구조를 바꿔야 하면 **먼저 ARCHITECTURE.md를 갱신**한 뒤 코드.
4. **검증**: 아래 "완료 정의"를 모두 만족하는지 직접 실행해 확인(typecheck·lint·관련 테스트, 필요 시 dev 기동·수동 확인). 추측으로 통과 처리 금지.
5. **반영**: AC 충족 시 `TASK.md`의 해당 항목을 `[x]`로 변경.
6. **커밋**: 작은 단위로 커밋(아래 컨벤션). 그리고 1번으로 돌아간다.
7. 막히면 "정지·질문 규칙"을 따른다.

## 완료 정의 (Definition of Done — 작업 레벨)
- 작업의 AC를 충족한다.
- `npm run typecheck` 와 `npm run lint` 통과.
- 분석/계산·파서 등 로직 변경은 **단위 테스트 추가·통과**.
- 신규/변경 테이블은 **RLS 정책과 함께** 마이그레이션에 포함.
- 비밀키(`SUPABASE_SERVICE_ROLE_KEY`, `ANTHROPIC_API_KEY`)가 클라이언트 번들·`NEXT_PUBLIC_*`에 노출되지 않음.
- 사용자에게 보이는 텍스트는 한국어 기본(국제화 키 사용), 빈/오류 상태 카피 포함.

## 절대 규칙 (위반 시 작업 중단)
- **보안**: service_role·Anthropic 키는 서버 전용. AI 호출은 `/api/ai/*` 서버 라우트만. 원본 측정 데이터·PII를 외부 API에 직접 보내지 말고 `lib/ai/digest.ts`의 요약만 전송.
- **테넌시**: 모든 데이터 접근은 `org_id` + RLS로 격리. 민감 쓰기는 service_role을 쓰되 요청자의 멤버십·역할을 코드로 재검증.
- **데이터 무결성**: 업로드 파싱은 트랜잭션. 부분 저장 금지.
- **계산의 단일 출처**: 통계/Cp/Cpk/이상치는 ARCHITECTURE의 정의를 따르고 RPC로 계산. 0으로 나누기·n<2·규격 없음은 null 가드.
- **문서 동기화**: 스키마·구조 변경 시 ARCHITECTURE.md를, 범위 변경 시 PRD.md를 먼저 갱신.
- **금지**: 임의 라이브러리 대량 추가, 인증/RLS 우회, 테스트 삭제로 통과 위장, `--no-verify` 강제 커밋.

## 정지·질문 규칙 (다음 경우에만 멈추고 질문)
- 외부 비밀(실제 Supabase/Anthropic 키, OAuth 클라이언트) 발급이 필요할 때 → 무엇을 어디 환경변수에 넣어야 하는지 안내하고 대기.
- 결정이 PRD의 "확정 필요(A1~A5)" 가정을 바꿔야 할 때.
- 파괴적 작업(데이터 삭제, 마이그레이션 되돌리기, 비용 발생).
그 외에는 합리적 기본값으로 진행하고, 선택한 가정을 커밋 메시지/PR 설명에 1줄로 남긴다.

## 커밋·브랜치 컨벤션
- Conventional Commits: `feat:`, `fix:`, `chore:`, `refactor:`, `test:`, `docs:`. 예: `feat(ingest): xlsx 파서 및 컬럼 매핑 (TASK 4.1)`
- 커밋 본문에 처리한 TASK 번호와 (있다면) 채택한 가정 1줄.
- 큰 단계(Phase) 종료 시 변경 요약을 출력.

## 진행 보고 (각 작업 종료 시 짧게)
- 완료한 TASK 번호 / 만든·바꾼 파일 / 실행한 검증 / 다음 작업 1줄.

## 시작
