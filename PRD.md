## 1. 개요

| 항목 | 내용 |
|---|---|
| 한 줄 정의 | 측정장비(이미지측정기 등)가 만든 치수 측정 데이터를 업로드하면, 제품 형상·공정능력(Cp/Cpk)·이상치를 자동 분석하고, 자연어로 질의할 수 있는 멀티테넌트 SaaS |
| 1차 도메인 | 치과재료(거타퍼차 콘 등) 및 정밀 부품 제조 중소기업의 품질관리(QC) |
| 시드 고객 | 다이아덴트(치과재료) — 기존 LM-X 100TL 이미지측정기 측정 데이터 보유 |
| 차별점 | (1) 측정값을 실제 제품 형상으로 복원해 현장 직관성 확보, (2) 규격 공차 기반 Cp/Cpk 자동 판정, (3) 데이터에 한국어로 질문하는 AI 분석가 |
| 비차별 영역(out) | 측정장비 펌웨어 연동, MES/ERP 풀스택 대체, 통계 패키지(Minitab) 완전 대체 |

### 핵심 가정 (확정 필요)
- A1. 데이터 소스는 측정장비(LM-X 100TL 이미지측정기)가 내보낸 **Excel(.xlsx)** 파일이며, 장비 직접 연동은 v2 이후. **현시점 업로드 기반으로 설계.**
- A2. 측정 포인트 수와 라벨은 제품마다 다르다. **시드 데이터 기준: GP 콘 = D0~D24(직경 25개) + 전장 1개 = 26개 포인트.** 스키마는 가변 포인트를 전제.
- A2-1. **파일 구조가 고정**: 1파일=1제품, 시트=사이즈(시트명 `"{제품} #{사이즈}"`), Row2~3=헤더, Row4+=데이터, 하단=요약통계행. 규격 공차는 파일에 미포함.
- A3. 합격/불합격 판정에 필요한 규격 공차(nominal/USL/LSL)는 고객이 직접 입력하거나 CSV로 등록한다.
- A4. 멀티테넌트(조직 단위 격리). 단일 조직 셀프호스팅은 동일 코드로 커버.
- A5. 과금은 MVP 범위 밖(조직/시트 수 모델을 v1.1에서 도입).

## 2. 사용자와 핵심 시나리오

### 페르소나
- **QC 담당자(Analyst)** — 매일 측정 파일(.xlsx, LM-X 100TL 출력)을 업로드하고 결과를 확인. 비개발자. 가장 빈번한 사용자.
- **QC 팀장/관리자(Admin)** — 제품·규격을 관리하고 조직원을 초대. 추세와 보고서를 본다.
- **경영진/뷰어(Viewer)** — 읽기 전용 대시보드와 보고서만 본다.
- **조직 소유자(Owner)** — 결제·조직 설정·삭제 권한.

### 대표 시나리오 (User Journey)
1. Owner가 가입 → 조직 생성 → 팀원을 이메일로 초대.
2. Admin이 제품(`Confirm Fit GP`)과 사이즈(F1/F2/F3/FX/FXL) 등록, 각 포인트의 규격 공차 입력.
3. Analyst가 측정 Excel을 업로드 → 컬럼 매핑 확인 → 저장(배치 생성).
4. 시스템이 배치를 파싱·집계하고 형상/SPC/이상치를 계산.
5. Analyst가 대시보드에서 형상·테이퍼 산포·Cp/Cpk·불량 목록을 확인.
6. Analyst가 AI 분석가에게 "팁 직경이 가장 불안정한 사이즈는?"이라고 질문 → 실제 통계 기반 답변.
7. Admin이 PDF 보고서를 내보내 경영진에 공유.

## 3. 기능 요구사항

표기: `[MVP]` 출시 필수 / `[v1]` 정식판 / `[v2]` 이후. ID는 `TASK.md`가 참조.

### F-1. 인증·조직 (Auth & Tenancy)
- F-1.1 `[MVP]` 이메일/매직링크 + Google OAuth 로그인 (Supabase Auth).
- F-1.2 `[MVP]` 첫 로그인 시 조직 생성 온보딩. 사용자당 다중 조직 소속 허용.
- F-1.3 `[MVP]` 역할: `owner`/`admin`/`analyst`/`viewer`. 역할별 권한은 RLS로 강제.
- F-1.4 `[v1]` 이메일 초대·수락 흐름, 멤버 관리 화면.

### F-2. 제품·규격 관리 (Master Data)
- F-2.1 `[MVP]` 제품 CRUD(이름, 코드, 설명).
- F-2.2 `[MVP]` 제품 사이즈/변형 CRUD.
- F-2.3 `[MVP]` 측정 포인트 정의: 라벨·순서. **시드 기준: D0~D24(직경 25개) + 전장(1개) = 26포인트.** 제품 사이즈별로 가변.
- F-2.4 `[MVP]` 포인트별 규격: nominal / USL / LSL. 직접 입력 + CSV 일괄 등록.
- F-2.5 `[v1]` 규격 이력(버전) 관리 — 어떤 배치가 어떤 규격으로 판정됐는지 추적.

### F-3. 데이터 수집 (Ingestion)
- F-3.1 `[MVP]` Excel(.xlsx) 업로드. **1파일 = 1제품, 시트 = 사이즈**(시트명 `"{제품} #{사이즈}"` 패턴). CSV는 단일 사이즈로 취급.
- F-3.2 `[MVP]` 컬럼 자동 인식(Row 2~3 헤더 탐지: "No"/D0~D24/"전장") + 사용자 매핑 확인 UI. 장비명은 C4 셀에서 추출.
- F-3.3 `[MVP]` 파싱 → 시트당 배치(batch) + 측정(measurement) + 포인트(measurement_point) 저장. 요약 통계행(평균값/최소값 등 D열이 비정수)은 자동 제외.
- F-3.4 `[MVP]` 유효성 검사: 빈 행/비수치/요약행 제거, 처리 리포트(총 행·유효 행·제외 행·제외 사유) 표시.
- F-3.5 `[MVP]` 배치 삭제: 배치 목록에서 선택 → 확인 다이얼로그 → cascade 삭제(measurements, measurement_points 포함). analyst 이상 권한. 중복 업로드 시 삭제 후 재업로드로 대응.
- F-3.6 `[v1]` 원본 파일을 Storage에 보관, 재처리·다운로드.
- F-3.7 `[v2]` 폴더/스케줄 자동 수집, 측정장비 직접 연동, CSV 지원 확장.

### F-4. 분석 엔진 (Analytics)
- F-4.1 `[MVP]` 포인트별 기술통계: n, mean, std, min, max, range.
- F-4.2 `[MVP]` 공정능력: Cp, Cpk (규격 등록 시). 규격 없으면 평균±3σ 관리한계로 대체 표기.
- F-4.3 `[MVP]` 형상 복원: 포인트 평균으로 콘 단면 프로파일 생성.
- F-4.4 `[MVP]` 이상치 검출: 규격 이탈(out-of-spec) + 통계 이탈(±kσ, 기본 4σ). 사유 포함.
- F-4.5 `[v1]` 배치 간 추세: 시간축 mean/Cpk 추이, 관리도(X̄-R).
- F-4.6 `[v1]` 분석 결과 캐싱·증분 갱신(대용량 대비).

### F-5. 대시보드·시각화 (Dashboard)
- F-5.1 `[MVP]` 사이즈 선택 → 형상 프로파일(캘리퍼 호버), 테이퍼+산포밴드(±3σ), 팁 직경 히스토그램, 전장 관리도, KPI 카드.
- F-5.2 `[MVP]` 불량 스크리닝 목록(샘플·포인트·이탈 σ·사유).
- F-5.3 `[v1]` 사이즈 다중 비교, 배치 비교, 추세 뷰.
- F-5.4 `[v1]` 반응형/다크모드/접근성(키보드·명도대비).

### F-6. AI 분석가 (AI Analyst)
- F-6.1 `[MVP]` 자연어 질의 → 서버에서 Anthropic API 호출 → 통계 요약 기반 한국어 답변.
- F-6.2 `[MVP]` **API 키는 전적으로 서버 측 보관.** 클라이언트 노출 절대 금지.
- F-6.3 `[MVP]` 대화 이력 저장(조직·배치 컨텍스트 포함), 예시 질문 제공.
- F-6.4 `[v1]` 답변에 근거 차트/숫자 인용 링크. 토큰·요청 레이트 제한.
- F-6.5 `[v2]` NL2SQL: 자연어를 조직 데이터에 대한 안전한 읽기 전용 SQL로 변환·실행(읽기 전용 뷰·권한 격리 전제).

### F-7. 보고서·내보내기 (Reporting)
- F-7.1 `[MVP]` 현재 대시보드 PNG/PDF 내보내기.
- F-7.2 `[v1]` 배치 요약 보고서 템플릿(로고·규격·판정 요약), 정기 메일 발송.

### F-8. 알림 (Notifications) `[v1]`
- F-8.1 Cpk 임계 미만/이상치 급증 시 이메일·인앱 알림.

## 4. 비기능 요구사항 (NFR)

- **보안**: 모든 테이블 RLS로 조직 격리. 비밀키는 서버 환경변수만. 업로드 파일은 조직 스코프 Storage 정책. OWASP Top 10 점검.
- **성능**: 배치당 ≤ 50k 측정 포인트 처리 ≤ 5초(서버 집계 RPC 기준). 대시보드 LCP ≤ 2.5s.
- **신뢰성**: 파싱 실패 시 부분 저장 금지(트랜잭션). 명확한 오류 메시지.
- **국제화**: 한국어 기본, 영어 토글(`ko`/`en`). 숫자/단위 일관 표기.
- **접근성**: WCAG 2.1 AA 지향(키보드 내비게이션, 포커스 가시성, 명도대비).
- **관측성**: 서버 로그·에러 추적(Sentry 권장), 핵심 이벤트 분석.
- **테스트**: 분석 계산(통계/Cpk/이상치)은 단위 테스트 필수. E2E로 업로드→대시보드 경로 1개 이상.

## 5. 기술 스택 (요약 — 상세는 ARCHITECTURE.md)

- **Frontend/Hosting**: Next.js (App Router, TypeScript) on **Vercel**, Tailwind CSS, shadcn/ui, Recharts.
- **Backend/Data**: **Supabase** (Postgres + Auth + Storage + RLS + RPC/Edge Functions).
- **AI**: Anthropic Claude API (서버 라우트 경유).
- **품질**: ESLint, Prettier, Vitest(단위), Playwright(E2E), GitHub Actions(CI).

## 6. 성공 지표 (KPI)

- 도입: 시드 고객(다이아덴트)에서 주간 활성 업로드 ≥ 5회/주, 4주 이상 지속.
- 가치: "이상치를 수동 대비 더 빨리 찾는다"는 정성 피드백 + 업로드→인사이트 시간 ≤ 3분.
- 신뢰: 분석 계산 정확도 — 단위 테스트 통과율 100%, 고객 규격과 판정 불일치 0건.
- 확장: 2번째 조직(예: 창명제어기술류 제조 SME) 온보딩 성공.

## 8. 릴리스 기준 (Definition of Done — 제품 레벨)
1. 시드 고객이 자기 데이터로 업로드→대시보드→AI 질의→보고서를 막힘 없이 수행.
2. 모든 테이블 RLS 적용 및 교차 조직 접근 차단 테스트 통과.
3. 분석 계산 단위 테스트와 업로드 E2E 통과, CI 그린.
4. Vercel + Supabase 프로덕션 배포, 비밀키 누출 없음(자동 점검 포함).
